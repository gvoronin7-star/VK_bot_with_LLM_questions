## 🎯 Функционал

- **Асинхронная архитектура** на базе `vkbottle`
- **Интеграция с LLM** через ProxyAPI (GPT-4.1)
- **Сохранение контекста** диалога в памяти и файле
- **Ограничение контекста** 4000 токенами (FIFO)
- **Отчётность** о работе в формате Markdown
- **Логирование** запросов, ответов и ошибок
- **Поддержка одного пользователя** без механизмов изоляции

## 📦 Требования

- Python 3.10+

## 🔧 Установка

### 1. Создать сообщество ВКонтакте

1. Перейдите в [VK](https://vk.com/)
2. Создайте новое сообщество (группу)
3. Убедитесь, что вы **администратор** группы

### 2. Включить сообщения в группе

1. Откройте группу
2. **Управление → Сообщения**
3. Включите **"Сообщения сообществ"**
4. Сохраните изменения

### 3. Включить Long Poll API

1. **Управление → Дополнительно → Работа с API**
2. Вкладка **Long Poll API**
3. Выберите **"Включено"**
4. Сохраните

### 4. Создать токен доступа

1. **Управление → Дополнительно → Работа с API → Ключи доступа**
2. Нажмите **"Создать ключ"**
3. Выберите права доступа:
   - ✅ `messages` — доступ к сообщениям
   - ✅ `groups` — управление сообществом
   - ✅ `manage` — администрирование
4. Скопируйте полученный токен

### 5. Настроить ProxyAPI

Получите ключ доступа к ProxyAPI у вашего провайдера LLM.

## ⚙️ Настройка

### 1. Создать файл `.env`

Скопируйте `.env.example` в `.env`:

```bash
cp .env.example .env
```

Откройте файл `.env` и заполните значения:

```env
# Ключ доступа сообщества
VK_ACCESS_TOKEN=your_vk_access_token_here

# ID сообщества (например, 123456789 из vk.com/club123456789)
VK_GROUP_ID=your_group_id_here

# Версия VK API
API_VERSION=5.199

# LLM ProxyAPI Configuration
PROXYAPI_URL=https://proxy-api.example.com/v1/chat
PROXYAPI_KEY=your_proxyapi_key_here

# LLM Model Settings
LLM_MODEL=gpt-4.1
TEMPERATURE=0.7
MAX_TOKENS=1000

# Context Settings
CONTEXT_FILE=data/context.json
MAX_TOKENS_IN_CONTEXT=4000
```

### 2. Получить ID группы

- Если группа по адресу `vk.com/club123456789` → ID = `123456789`
- Если группа по адресу `vk.com/mygroup` → ID = `-111111111` (со знаком минус)

## 💬 Команды

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и начало диалога |
| `/clear` | Очистка контекста диалога |
| `/help` | Показать справку с описанием команд |
| `/report` | Показать отчёт о работе бота |
| `/settings` | Показать текущие настройки |
| `/settemp <0.1-1.0>` | Установить температуру (например: `/settemp 0.9`) |
| `/settokens <100-4096>` | Установить лимит токенов (например: `/settokens 2000`) |
| `/rate creative` | Оценить эффект как креативный |
| `/rate concise` | Оценить эффект как лаконичный |

### Настройка температуры и токенов

Вы можете настроить параметры LLM прямо из бота:

```
/settemp 0.9        # Увеличить креативность
/settokens 2000     # Увеличить длину ответа
```

Настройки сохраняются для каждого пользователя индивидуально.

### Примеры использования

```
Пользователь: Привет!
Бот: 👋 Привет! Я бот с поддержкой LLM GPT-4.1. ...

Пользователь: Расскажите о машинном обучении
Бот: Машинное обучение — это область искусственного интеллекта...

Пользователь: /clear
Бот: 🧹 Контекст диалога очищен. Давайте начнём заново!
```

## 📊 Отчётность

Отчёт формируется автоматически после каждого запроса к LLM в файле `REPORT.md`.

### Пример отчёта

```markdown
# Отчёт о работе бота

| Модель | Temperature | Max Tokens | № прогона | Эффект | Использованные токены | Ориентировочная стоимость |
|--------|-------------|------------|-----------|--------|----------------------|--------------------------|
| gpt-4.1 | 0.7 | 1000 | 1 | balanced | 150 | $0.0015 |
| gpt-4.1 | 0.7 | 1000 | 2 | creative | 230 | $0.0023 |
| gpt-4.1 | 0.7 | 1000 | 3 | concise | 80 | $0.0008 |
```

### Автооценка эффекта

- **creative** — Высокая сжатость и лексическое разнообразие
- **concise** — Низкая сжатость, высокое разнообразие
- **balanced** — Сбалансированные метрики

### Ручная оценка

Используйте команду `/rate creative` или `/rate concise` для ручной оценки эффекта.

## 🆘 Решение проблем

### Бот не отвечает

1. Проверьте токен в `.env`
2. Убедитесь, что Long Poll включён
3. Проверьте права доступа (должен быть `messages` и `manage`)
4. Проверьте настройки ProxyAPI

### Ошибка подключения к ProxyAPI

1. Проверьте `PROXYAPI_URL` и `PROXYAPI_KEY` в `.env`
2. Убедитесь, что ключ активен
3. Проверьте сетевые настройки (прокси, фаервол)

### Контекст не сохраняется

1. Проверьте права записи в директорию `data/`
2. Убедитесь, что файл `data/context.json` не заблокирован

### Бот зацикливается

1. Остановите все Python процессы
2. Убедитесь, что нет нескольких запущенных экземпляров бота
3. Перезапустите бота

### Ошибки токенов

1. Проверьте `MAX_TOKENS_IN_CONTEXT` в `.env`
2. Убедитесь, что контекст не превышает лимит
